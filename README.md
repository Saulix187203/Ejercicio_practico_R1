# Ejercicio_practico_R1
Aplicar los conceptos de análisis asintótico, casos (mejor, promedio y peor) y trade-off tiempo–espacio, implementando y comparando dos algoritmos de ordenamiento bajo un flujo de trabajo controlado con Git y GitHub.

# Instrucciones:
1. Crea una carpeta de proyecto llamada analisis_algoritmos e inicializa un repositorio con git
init.
2. Implementa el algoritmo Bubble Sort en un archivo bubble_sort.py.
3. Crea comparacion.py para comparar los tiempos de ejecución de Bubble Sort y sorted() con
listas de 100, 500, 1000 y 5000 elementos.
4. Mide los tiempos de ejecución con time.perf_counter() y guarda los resultados en una tabla.
5. Documenta el proyecto en un archivo README.md con tu nombre, objetivos y explicación
breve.
6. Registra al menos tres commits descriptivos: implementación inicial, comparación, resultados
finales.
7. Crea un repositorio en GitHub llamado Analisis-Algoritmos-NombreApellido y sube tu
código.
8. Incluye los archivos: README.md, bubble_sort.py, comparacion.py, analisis.md y
resultados.txt (opcional).
9. Envía el enlace del repositorio al ingeniero antes de la fecha límite.

# Conclusión esperada:
El estudiante debe reconocer la diferencia entre el crecimiento
cuadrático (O(n²)) del algoritmo burbuja y el comportamiento casi lineal (O(n log n)) del método
sorted(), interpretando los resultados con base en el análisis asintótico.
